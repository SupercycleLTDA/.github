# O que acontece com toda essa conversa?

É importante saber o que fazemos com todas as contribuições que recebemos de ideias, perguntas,
votos e discussões resultantes.

Quando temos certeza de que faz sentido adicionar uma ideia ao software, criamos uma "User Story" com base na ideia.

Essa User Story é usada para definir exatamente o que precisamos fazer para atingir o objetivo que a Story descreve.

É por isso que uma Story segue um formato padrão na forma como é escrita:

* Quem conta a Story (Perspectiva/Pessoa)
* O que é desejado (Funcionalidade/Necessidade)
* Por que isso é útil (Razão/Finalidade)

Exemplo:

Como Coach, quero poder mudar a cor das luzes no estúdio para poder influenciar melhor o ambiente do Ride diretamente.

* Perspectiva: O Coach
* Funcionalidade: Altere a cor das luzes do Studio
* Motivo: Para melhor influenciar o ambiente do Ride

Uma vez que decidimos que uma ideia se torna uma Story, ela será adicionada à lista de Stories aqui
👉🏼[User Stories](https://github.com/orgs/SupercycleLTDA/projects/1)

A partir daí, decidiremos qual Story será incluída em uma iteração de engenharia 🤓

## Iteration (Iteração)

Há muitas maneiras pelas quais os engenheiros de software podem lidar com a escolha de quando uma Story é
adicionada ao software. Como achamos que faz mais sentido que quem usa o software escolha o que ele pode fazer,
montamos um processo para orientar isso:

- Todos os Coaches são convidados para as discussões, fazer perguntas e criar/participar de enquetes (veja o explicador
  acima)
- O tópico de discussão que recebe mais votos é transformado em uma Story
- Em nossa reunião semanal, vemos o que na lista de Stories precisa de atenção
- Escolhemos 1 Story para desenvolver na próxima iteração

Dessa forma, mantemos um ritmo confortável ao adicionar e alterar funcionalidades.
Isso nos dá tempo para pensar e responder aos tópicos enquanto perguntamos para decidir o que construir a cada semana.

Como em tudo o que fazemos, este processo é um ponto de partida, quando as coisas não funcionam bem na prática,
podemos fazer mudanças que consideramos úteis.

Este processo de melhoria é chamado de Iteração. <br>
E essas iterações continuam indefinidamente.

Em outras palavras, se continuarmos iterando por tempo suficiente, tudo ficará perfeito! 😁
